MyFree 导航页 - 群晖/PHP 简易动态版
这是一个轻量级、响应式的主页导航系统，专为群晖 (Synology) 或任何支持 PHP 的服务器环境设计。

✨ 特色功能
智能后端：基于 PHP + JSON，无需数据库，部署即用。

自适应 UI：支持手机/桌面端响应式布局。

安全认证：自带管理员登录系统，支持在线编辑、添加、删除导航链接。

私密存储：针对特定分类支持密码锁定。

智能 Icon：支持自动获取 Favicon，支持手动设置 Emoji。

动态环境：自带天气预报组件与实时日期显示。

🚀 部署步骤
1. 前置准备
确保你的服务器环境已安装 PHP (推荐 PHP 7.4 或更高版本)。

确保 PHP 拥有对项目目录的 写入权限（因为需要实时保存 links.json）。

2. 安装部署
下载代码：将本项目下载并解压到你的 Web 服务器根目录（例如 /web/myfree）。

设置权限：确保 links.json 文件存在，且 Web 服务器用户（如 http）对该目录拥有读写权限。

提示：如果首次运行该文件不存在，系统会自动创建一个默认数据。

访问项目：通过浏览器访问你的网站地址。

3. 修改管理员账号 (非常重要)
为了安全，请务必修改默认的 ADMIN_USER_HASH 和 ADMIN_PASS_HASH。

由于代码中使用 password_hash 加密，请在本地环境通过以下简单的 PHP 脚本生成对应的哈希值：

PHP
<?php
// 生成账号和密码的哈希，将输出结果替换代码中的定义
echo password_hash("你的用户名", PASSWORD_DEFAULT) . "\n";
echo password_hash("你的密码", PASSWORD_DEFAULT);
?>
将生成的两行字符串替换代码顶部的 ADMIN_USER_HASH 和 ADMIN_PASS_HASH 常量。

⚙️ 后台管理说明
点击页面右上角的 "⚙️ 管理员登录"。

输入设置的账号密码后，页面会自动切换至管理模式。

在管理模式下，你可以：

点击分类卡片上的 ➕ 添加新链接。

直接拖动排序或编辑链接信息。

记得点击页面底部的 "确认应用到群晖" 按钮以保存数据。


🚀 群晖 (Synology) 部署指南
第一步：准备 Web 环境
安装套件：打开群晖的【套件中心】，搜索并安装以下套件（如果已安装请跳过）：

Web Station（管理网页服务的核心）

PHP（建议安装 PHP 7.4 或 8.x 版本）

Apache HTTP Server (推荐 2.4 版本)

创建目录：打开【File Station】，在 web 共享文件夹下创建一个新文件夹，命名为 myfree（或其他你喜欢的名字）。

第二步：上传文件
将你的 PHP 文件代码复制并保存为 index.php。

将 index.php 上传到群晖的 /web/myfree 目录下。

第三步：配置 Web Station
打开【Web Station】。

设置 PHP 配置：

进入【PHP 设置】，选择你安装的 PHP 版本，点击【编辑】 -> 【核心设置】。

确保 open_basedir 包含了你的 web 目录（通常默认配置即可，如果遇到报错，请将 /volume1/web 加入其中）。

配置服务门户 (Web 服务)：

进入【网页服务门户】（或旧版 Web Station 的【虚拟主机】）。

点击【新增】 -> 【服务门户】 -> 【虚拟主机】。

主机名：你可以填入你的自定义域名或留空（留空则通过 IP 访问）。

文档根目录：选择刚刚创建的 /web/myfree 文件夹。

HTTP 后端服务器：选择 Apache 2.4。

PHP 配置：选择你在第二步中确认的 PHP 版本。

第四步：设置读写权限 (最重要的一步)
因为该脚本需要实时写入 links.json 文件，如果权限不足，你将无法在网页上进行修改和保存。

在【File Station】中，右键点击 /web/myfree 文件夹，选择【属性】。

点击【权限】选项卡 -> 【新增】。

用户/组选择 http（群晖的系统 Web 用户）。

勾选【读取】和【写入】权限，点击【确定】并勾选【应用到此文件夹、子文件夹及文件】。

第五步：测试访问
打开浏览器，访问 http://你的群晖IP/myfree/。

如果一切正常，你应该能看到导航页加载出来。

点击页面右下角的【⚙️ 管理员登录】，测试能否正常进入管理模式。

🛠️ 故障排除 (FAQ)
页面显示空白？：请检查【Web Station】中的 PHP 错误日志，确认是否缺少 PHP 扩展（该脚本通常不需要特殊扩展，但请确认 PHP 已启用）。

无法保存链接？：这 90% 是因为 links.json 没有写入权限。请严格检查第四步的 http 用户权限。

HTTPS 问题：如果你通过域名加 SSL 访问，请确保在群晖的【控制面板】 -> 【登录门户】 -> 【高级】 -> 【反向代理】中正确配置了证书，或者直接在 Web Station 门户设置中绑定证书。

💡 进阶建议：安全性防护
由于这是一个带有管理后台的脚本，强烈建议：

自定义路径：不要直接使用 /web/myfree，可以将文件名改得复杂一些（例如 /web/a1b2c3d4/），防止被扫描。

增加反向代理验证：如果在群晖外网直接访问，建议在群晖的反向代理中添加【访问控制配置】，只允许你的指定 IP 访问此端口。

禁用目录索引：确保 Web Station 的设置中禁用了目录列表显示功能。
